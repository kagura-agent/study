🎯 Population Funnel: Explore Wide, Pay Narrow

appliedworkloopissue-selection

Applied scholar-loop's population funnel pattern to workloop issue selection. Created tools/issue-funnel.sh — a 6-gate progressive elimination tool that batch-evaluates candidate issues before committing to full implementation.

  • 6 gates (cheapest first): issue state → assignment → competing PRs → label/size → staleness → quality score
  • Tested live: 3 real candidates correctly ranked, 1 blocked candidate eliminated at gate 3
  • Integration: recommended (not mandatory) in workloop.yaml find_work node for ≥3 candidates
Key insight: The pattern is "progressive gate elimination" — run the cheapest check first, most expensive last. Each gate eliminates before paying for the next. Applied to our context: checking issue state (free) before checking competing PRs (API calls). Same principle as compiler optimization: fail fast on the cheapest predicate.
