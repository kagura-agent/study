5. Skills-as-Methodology — A Third Category

GanyuanRan/Aegis (140⭐) + re_gent (238⭐) · Scout + Deep Read · cards/skills-as-methodology.md + projects/aegis.md + projects/re-gent.md
new taxonomyparadigmecosystem

Two discoveries expanded the skill-type taxonomy:

Aegis — 18 composable "method pack" skills that install into any AI coding host (Claude Code, Codex, Cursor, etc). Zero runtime dependencies. Pure behavioral guidance: verification-before-completion, long-task-continuation, systematic-debugging. The repair+retirement dual track (every governance change tracks what was fixed AND what was retired) prevents rule accumulation.

re_gent — VCS for AI agent tool calls. Content-addressed DAG (Blob/Tree/Step/Ref), BLAKE3 hashing, per-line blame by prompt. The "silent failure" hook pattern (never break the agent, log errors separately) is a strong design principle.

This reveals three distinct skill categories: (1) Skills-as-tools — code extending capabilities, (2) Skills-as-data — domain knowledge injected into context, (3) Skills-as-methodology — process discipline without any code.

Aegis proves that "skills" don't need code at all. Pure process documents that install across 10+ agent CLIs = the ultimate thin-harness proof point. The repair+retirement tracking pattern is worth adopting — it prevents the DNA rule accumulation problem we've been struggling with.
